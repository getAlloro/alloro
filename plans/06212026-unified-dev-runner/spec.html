Why

In the main alloro repo, starting local dev takes three terminals: backend (npm run dev), worker (npm run dev:worker), and frontend (cd frontend && npm run dev). The sibling repos alloro-chat and alloro-os already start everything with one root npm run dev via concurrently. This brings the same one-command experience here.

What

Root npm run dev starts backend API, worker, and frontend together as three labeled, colored streams in one terminal. The previous backend-only behavior stays available as npm run dev:api. Done when a single npm run dev boots all three processes.

"dev":        "concurrently -n api,worker,web -c blue,yellow,green \"npm:dev:api\" \"npm:dev:worker\" \"npm:dev:web\"",
"dev:api":    "tsx watch src/index.ts",            // was "dev"
"dev:worker": "tsx watch src/workers/worker.ts",   // unchanged
"dev:web":    "npm --prefix frontend run dev"       // new

Context

Relevant Files

  • package.json (root) - scripts + devDependencies edited here
  • src/index.ts - backend entry, run by dev:api (verified present)
  • src/workers/worker.ts - worker entry, run by dev:worker (verified present)
  • frontend/package.json - has dev (= vite), run by dev:web

Patterns To Follow

  • Mirror the sibling root scripts in ~/Desktop/alloro-chat/package.json and ~/Desktop/alloro-os/package.json: concurrently -n <names> -c <colors> with npm:<script> shorthands.
  • Same dependency the siblings use: concurrently ^9.1.2 as a devDependency.

Reference: ~/Desktop/alloro-os/package.json line 16 (root dev orchestrator). This repo drops the siblings' dev:ws stream (no WS server) and the compose:up prefix (no infra/docker-compose).

Code Constitution: not materially applicable. The change touches only root package.json dev tooling — no src/ backend or frontend/ application code, so no §N.M architecture Article is engaged.

Constraints

Must

  • Preserve backend-only startup under a named script (dev:api).
  • Use concurrently to match sibling DX (unified logs, labels, single Ctrl-C).

Must Not

  • No compose:up / docker step — this repo has no local infra/.
  • Do not change ports, the backend, the worker, or any application code.
  • Do not touch frontend/package.json scripts.

Out Of Scope

  • WS server stream (none in this repo).
  • Local Postgres/Redis provisioning — dev still relies on .env pointing at reachable services, same as today.

Risk

Level: 2 (Concern — changes what an existing command does).

Risks Identified

  • Behavior change of npm run dev: it no longer means "backend only." Mitigation: backend-only is preserved verbatim as npm run dev:api; the change is the explicit ask ("implement the same" as the siblings).
  • Worker needs Redis at runtime (BullMQ): a Redis-less environment shows a worker-stream error. Mitigation: pre-existing — identical to running npm run dev:worker today; this change only co-launches it, it adds no new requirement.

Blast radius: root package.json only. No CI workflow, README, or AGENTS.md references npm run dev; the sole mentions are in historical plans/ specs (immutable records). Developer muscle memory is the only real consumer.
