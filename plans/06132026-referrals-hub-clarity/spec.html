Why

Plan 2 of the dashboard-revamp feedback round — covers PDF item #3 ("Referrals Hub"). The Referrals Hub (/pmsStatistics, PmsHubSurface) shows a "This mo" tile that never names the month, a MONTH/QTR/YTD toggle that only re-scopes 2 of 4 tiles, a YTD figure stranded in a separate tile away from the monthly production it should sit beside, and no one-line read on whether the trend is good or bad. Consumes the Plan 0 primitives (shared PeriodToggle, formatDataMonth, InsightCue) so this surface matches the new standard the other hubs adopt.

What

Done when:

  • #3.1 Named month: the period-scoped production + referrals tiles, when MONTH is active, read the actual latest uploaded month via formatDataMonth ("April 2026") — never "This mo". The referrals tile clarifies "Referrals · April 2026".
  • #3.2 Toggle drives every tile (or fixed ones are marked): the shared terracotta PeriodToggle replaces the local white-pill one. Labels: Month / QTD / YTD. Production + referrals re-scope on toggle. The Sources tile carries no per-period data (keyData has no per-month source counts) so it is explicitly marked all-time ("All-time" sub) rather than silently ignoring the toggle.
  • #3.3 YTD beside monthly: the standalone YTD tile is removed; the YTD production figure renders as a secondary line inside the period-production tile, directly beneath the period number — so monthly and YTD sit together. (Hidden when the active period already IS YTD, to avoid a duplicate.)
  • #3.4 InsightCue: one InsightCue below the chart, driven by latest-vs-previous production bucket of the active series (up → strong, down → spend time here, flat → holding). One line only.
  • eslint clean on the touched files (orchestrator runs authoritative tsc).

Context

Confirmed in code (the findings driving this plan)

  • Local PeriodToggle: PmsHubSurface.tsx lines 38–67 define a private white-active-pill toggle over Period = "MONTH"|"QTR"|"YTD". Plan 0's shared PeriodToggle (terracotta active) is label-agnostic and takes options=[{key,label,tooltip?}].
  • "This mo" mislabel: tile 1's label is periodCardLabel(period) → returns the bare string "This mo" / "This qtr" / "YTD" (pmsPeriod.ts line 172). No month name.
  • Only 2 of 4 tiles re-scope: the tiles array (lines 197–210) — tile 1 (production) + tile 2 (referrals) use scoped/hovered; tile 3 (Sources) is a fixed topSources.length; tile 4 is a fixed YTD (scopedTotals(monthlyData,"YTD")). The toggle changes the first two only.
  • Month key format: the orchestrator (PMSVisualPillars.tsx line 870) maps each month through formatMonthLabel → "Apr 2026" before passing monthlyData down. parseYM + formatDataMonth both handle that display-label form, so the latest key resolves to "April 2026" cleanly.
  • Latest-month anchor: scopedTotals/bucketByPeriod already sort ascending and anchor "current" to sorted.at(-1). The surface needs that same latest key for the named label — added as a tiny exported latestMonthKey helper in pmsPeriod.ts rather than re-sorting in the component.

Risks Identified

  • R1 — Concurrent edits (Level 2). Three other agents edit other surfaces simultaneously. Mitigation: stay strictly inside PmsHubSurface.tsx + pmsPeriod.ts (both exclusively this surface's); never touch shared primitives / index.css / statusRules.ts; run file-scoped eslint only (no full-project tsc — sibling in-flight edits would throw false errors; orchestrator runs authoritative tsc after).
  • R2 — Duplicate YTD (Level 1). When the active period is YTD, the primary number already is the YTD figure; also showing a "YTD" secondary would duplicate it. Mitigation: render the YTD secondary only when period !== "YTD" and not hovering (hover shows a specific bucket; YTD anchor stays meaningful there).
  • R3 — Trend direction source (Level 1). The InsightCue must reflect a real signal. Mitigation: derive from the LAST two production buckets of the ACTIVE series (trendData) so the cue tracks the toggled view; flat when <2 buckets or equal.

Blast radius: PmsHubSurface.tsx (Referrals Hub only) + pmsPeriod.ts (helpers consumed only by that surface; parseYM is also imported by utils/timeframe.ts — the new latestMonthKey is additive and does not touch parseYM). No other consumers.

Pushback

  • Sources genuinely has no per-period data, so "make the toggle drive EVERY tile" is not literally possible for it without backend work that is out of scope. The honest fix is the spec's second option — mark it all-time. Flagging so the owner knows Sources is intentionally fixed, not a missed wire-up.
  • Labels Month / QTD / YTD are mixed (one spelled noun, two abbreviations) per the explicit item-#3 instruction. This is intentional: the MONTH bucket is a single calendar month (named in the tile), whereas QTR/YTD are cumulative to-date measures that the locked standard abbreviates. Tooltips spell QTD/YTD out via TO_DATE.
