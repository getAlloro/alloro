Why

Two issues surfaced while verifying the audit work. One is a confirmed funnel-analytics bug; the other is a suspected route hang that needs a clean repro.

B1 — /api/leadgen/event returns 400 for 5 event types (CONFIRMED)

The backend gate isValidEventName accepts only keys in STAGE_ORDER. The frontend fires five names that aren't there, so each is rejected with 400 invalid_event_name and never recorded:

email_field_focused        ← in NON_STAGE_EVENTS, but isValidEventName ignores that set
email_field_blurred_empty  ← same
cta_clicked_create_account ← same
cta_clicked_strategy_call  ← same
audit_retried              ← in NO backend set at all (pure FE/BE drift)

The first four are already declared as legitimate interaction events in util.event-ordering.ts (NON_STAGE_EVENTS, with a "keep in sync with the leadgen-tool union" comment) — the validator just never consults that set. Net effect: every CTA click, email-field focus/blur, and audit retry is dropped from the funnel.

B2 — Cold ?audit_id= load appears to hang (SUSPECTED)

A fresh browser navigation to localhost:3002/?audit_id=<completed-audit> timed out in Playwright (navigate + snapshot), while the dev server returned 200 in ~25ms. Reading the mount path (App.tsx effects at 136/173/221) shows no synchronous loop — a cold completed-audit load routes input → dashboard cleanly. The likely cause is a heavy first paint (Google Maps iframe + many repeat:Infinity framer-motion animations) and/or accumulated headless-browser state — i.e. possibly a test artifact, not a product hang. This must be reproduced in a clean browser before any fix.

What

  • B1: isValidEventName accepts stage events and the declared interaction events; audit_retried is registered (as a non-stage interaction event). The five event types record again; no event 400s for being unknown.
  • B2: Reproduce in a clean browser session. If a real hang: isolate it (bisect the dashboard — Maps iframe, infinite animations, an effect) and fix. If a test artifact: document it and close, with any cheap hardening (e.g. lazy-mount the map) noted.

Risks & Mitigations

  • B1 over-permissive fix. Accepting the wrong set could let junk events in. Mitigation: accept exactly STAGE_ORDER ∪ NON_STAGE_EVENTS (+ audit_retried added to the interaction set); everything else still 400s. Unit-test both directions.
  • B1 silent drift recurs. Mitigation: a single shared accepted-set + a test asserting the FE union ⊆ BE accepted-set, so the next added event fails CI rather than prod.
  • B2 unknown root cause. Mitigation: time-boxed bisection (toggle the Maps iframe, then the infinite animations, then effects); if it only repros in headless with accumulated state, classify as artifact and stop.

Pushback

  • Ship B1 on its own — it's a confirmed funnel leak with a one-file fix and a clean test. Don't let it wait on B2's open-ended investigation.
