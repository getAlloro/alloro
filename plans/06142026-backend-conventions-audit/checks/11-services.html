Findings

Summary

Verdict: needs work. The top-level services/ tier is the architectural inverse of its own ai-seo-audit/ subfolder. The subfolder is the gold standard (consistent *Service naming, routes data through models/, uses db.transaction for multi-table writes). The flat top-level files do the opposite: they own inline DB access (12 / 42 files run db("table") directly — services holding SQL is the worst dim-7 case), skip the controller layer, and ship 150–230-line functions. The single most dangerous finding is unguarded multi-table writes with no transaction (referral rewards, instant-website creation). Severity: 5 must-fix · 8 concerns · 5 nitpicks. Most DB/console/validation items are repo-wide [systemic]; the unit's own headline is fat functions + transaction-less multi-table writes + naming drift broader than flagged.

Must-fix

  • referralReward.ts:35–231 — [dim 16] applyReferralReward + trackReferralSignup write across referrals, notifications, and read organizations with no transaction. A failure between the referrals.update({status:"converted"}) and the later update({status:"rewarded"}) (e.g. Stripe throws) leaves the referral half-rewarded and notifications inconsistent. Fix: wrap the status transitions + notification inserts in db.transaction; move all queries into a ReferralModel (this file alone holds 13 inline db() calls). Mirror ai-seo-audit/auditTargetExecutionService.ts:114, which does this correctly.
  • instantWebsiteGenerator.ts:510–565 — [dim 16] writes four tables in sequence (website_builder.projects, website_builder.pages, organizations, notifications) with no transaction. A crash after the project insert but before the page insert yields a project with no homepage. Fix: wrap the project+page+org writes in db.transaction; relocate the inserts to models.
  • referralReward.ts, behavioralIntelligence.ts, instantWebsiteGenerator.ts, configStore.ts, vocabularyAutoMapper.ts, firstPatientAttribution.ts, ozMoment.ts, businessMetrics.ts, ai-cost/service.ai-cost.ts — [dim 7] inline db("table") in services (12 files total, counts: referralReward 13, behavioralIntelligence 8, configStore 6, instantWebsiteGenerator 5). behavioralIntelligence.ts:177 and ai-seo-audit/organizationAuditContextService.ts:26,33 also use db.raw(...). Services holding SQL is the most severe form of the [systemic] dim-7 drift. Fix: push every query into the matching model (ReferralModel, SystemConfigModel, VocabularyConfigModel, etc.); services call models only.
  • surpriseFindings.ts:97 (generateSurpriseFindings 232 lines), instantWebsiteGenerator.ts:182 (buildHomepageSections 215), ozMoment.ts:118 (generateOzMoments 163), referralReward.ts:87 (applyReferralReward 153) — [dim 2] functions 3–5× the 50-line cap. Fix: extract named helpers (per-finding-type builders; per-section HTML builders already partially factored in instantWebsiteGenerator — finish the pattern; split the AI-prompt assembly from the orchestration in ozMoment).
  • configStore.ts:43–67 — [dim 8] setConfig and deleteConfig have no try/catch while getConfig/getAllConfig do — asymmetric error handling, and setConfig's read-then-insert/update is a non-atomic check (TOCTOU; concurrent writers can both hit the insert branch and collide on the key unique constraint). Fix: use a single upsert (onConflict("key").merge()) inside a model, wrapped in try/catch.

Concerns

  • routes/checkup.ts imports vocabularyAutoMapper, surpriseFindings, ozMoment, instantWebsiteGenerator, referralReward directly — [dim 5/6] the route skips the controller layer (Routes → Controllers → Services). Fix: route through a checkup controller; keep routes thin.
  • models/BehavioralEventModel.ts imports services/behavioralIntelligence.ts — [dim 5] layer inversion: a model depends on a service (models are the bottom layer and must not call up into services). Risks circular imports. Fix: invert the dependency — the service calls the model, never the reverse.
  • Hardcoded Claude model IDs as magic strings — [dim 11] reviewSentiment.ts:90 ("claude-haiku-4-5-20251001"), ozMoment.ts:202 (same), reviewThemeExtractor.ts:95 falls back to "claude-sonnet-4-6". Fix: name these in config/ (one source of truth for model selection). Note: ai-cost/pricing.ts model IDs are a legit pricing lookup table, not a violation.
  • $2000 referral reward hardcoded as -200000 cents — [dim 11] referralReward.ts:143,170, duplicated. Fix: a named constant (or pull from configStore, which already de-hardcodes org_monthly_rates).
  • Direct process.env.* reads bypassing config/ — [dim 11] instantWebsiteGenerator.ts:205, reviewThemeExtractor.ts:65,95, recipientSettingsService.ts:98–101, integrations/rybbitHarvestAdapter.ts:6–7, ai-seo-audit/externalEntitySearchService.ts:10. Contract says secrets/env are accessed via config/. Fix: centralize in a config module.
  • console.* in 10 files (referralReward 9, then instantWebsiteGenerator/firstPatientAttribution/ai-cost 2 each, +6 singles) — [systemic] dim-10 (Pino not installed). Fix: swap to the Pino logger once available; referralReward is the worst concentration.
  • Silent error-swallowing — [dim 8] empty catch {} / .catch(() => {}) at referralReward.ts:64,75,218,231, instantWebsiteGenerator.ts:565, vocabularyAutoMapper.ts:580, ozMoment.ts:~280. Some are intentional "non-blocking" by design, but several drop errors with no log at all (violates "if you catch it, log it"). Fix: at minimum log before swallowing.
  • Input validation absent on the route consumers of these services — [systemic] dim-12 (0/39 routes validated). Services trust raw caller input (org IDs, GBP category strings, checkup payloads). Fix: add validation/ schemas at the route layer.

Nitpicks

  • Naming drift is wider than the 5 files flagged — [dim 14] 12 top-level files lack PascalCase + Service suffix: the 5 listed (vocabularyAutoMapper, surpriseFindings, ozMoment, behavioralIntelligence, instantWebsiteGenerator) plus agentRegistry, businessMetrics, configStore, firstPatientAttribution, reviewSentiment, reviewThemeExtractor, referralReward. Fix: rename to VocabularyMapperService.ts etc. (large blast radius — many importers; do as a dedicated rename pass, not piecemeal).
  • ai-cost/service.ai-cost.ts — [dim 14] inverted/dotted filename; convention wants AiCostService.ts. integrations/harvest-registry.ts, harvest-types.ts use kebab-case where the rest of the tree is camelCase.
  • vocabularyAutoMapper.ts 585 lines & ai-seo-audit/types.ts 161 — [dim 1] over/near cap, but both are mostly static lookup data (category→preset map; type defs), not branching logic. Lower severity than a 585-line logic file; still a split candidate (move the CATEGORY_MAP table to a data module).
  • Pleasant surprises worth noting: zero commented-out code across the whole scope (dim-15 clean); surpriseFindings.ts is a correctly-pure transformer (no DB, returns data); ozMoment.ts AI call has a proper try/catch + buildTemplateFallback; the entire ai-seo-audit/ subfolder is consistent and uses models + transactions correctly.
