Why

The Team Members table lets an organization manage access, but its automatic column sizing wraps action labels even at the normal Settings width. Role editing also falls back to the operating system's native select menu, which does not match the surrounding interface and cannot provide controlled Alloro motion.

The same layout issues affect Pending Invitations. This work makes both tables stable and easier to scan without changing permissions, backend behavior, or the role-change commit flow.

What

Deliver polished semantic tables on /settings/users with explicit column contracts, single-line structural content, safe truncation for unbounded identity data, consistent action controls, and responsive horizontal overflow below the table's minimum width.

Replace both native role selects with one settings-local animated listbox. It must work by mouse and keyboard, render above table overflow without clipping, respect reduced-motion preferences, and preserve the existing Save/Cancel confirmation boundary. Mirror the default table presentation in Alloro Docs.

Interaction Contract

Tables

  • Use semantic table markup with explicit column widths and an approximately 820px minimum width for the member table.
  • Headers, role labels, dates, and action controls never wrap.
  • Long names and emails remain one line, truncate visually, and expose the full value by accessible title/label.
  • Below the minimum width, the table scrolls horizontally instead of crushing columns.
  • Use a subtle header band, restrained alternating rows, hover highlight, tabular dates, and compact consistent action buttons.

Role Editing

  • Change role enters staged edit mode and presents the custom role selector.
  • Selecting Viewer, Manager, or Admin changes local staged state only.
  • Save sends the existing mutation; Cancel restores the original role without a request.
  • The menu uses listbox semantics, Arrow Up/Down, Enter/Space, Escape, outside-click close, visible focus, and focus restoration.
  • The menu is portaled or otherwise placed in the top layer so table overflow cannot clip it; placement flips or clamps within the viewport.
  • Motion is a short fade/scale with chevron rotation and is reduced or removed when the user prefers reduced motion.

Invitation Selector

The invite modal reuses the same selector. Admins can choose Viewer, Manager, or Admin. Managers remain limited to Viewer and Manager. Existing backend enforcement stays authoritative.

Risk

Level: 2 — the visual change is contained, but the custom selector sits on a permissions surface and must preserve accessibility and staged-save behavior.

Risks Identified

  • Dropdown clipping: horizontal table overflow can clip an absolutely positioned menu. Mitigation: render the menu through a portal/top layer, measure the trigger, and clamp or flip placement within the viewport.
  • Accessibility regression: replacing a native select can lose keyboard and screen-reader behavior. Mitigation: implement the complete listbox contract, restore focus, honor reduced motion, and cover keyboard selection/escape in a focused test.
  • Accidental permission changes: an immediate mutation would make exploration destructive. Mitigation: selection remains staged; only Save calls the existing endpoint.
  • No-wrap overflow: forcing every value to full width would make the table unusable for long emails. Mitigation: no-wrap bounded structural fields, truncate unbounded identity fields, and scroll below the minimum width.
  • Extraction regression: splitting a 572-line component can break wizard demo data, role visibility, invitation constraints, or feedback. Mitigation: keep UsersTab as the state/orchestration owner, pass explicit typed props, and verify both admin and manager behavior.
  • Dirty docs checkout: direct edits would mix this work with Practice Hub changes. Mitigation: use a separate docs worktree and stage the replica explicitly.

Blast radius: UsersTab is consumed by UsersRoute, rendered in the client dashboard/Pilot flow, and fed by onboarding wizard demo data. Mutations continue to hit the same protected Settings routes and invalidate the same users query. The default-state visual mirror in Alloro Docs must change in step.

Pushback

  • Applying “no line breaks” literally to arbitrary names and emails would trade wrapping for overflow. The safer contract is single-line truncation with full-value access.
  • Changing the shared animated select is not justified here. It has five consumers and a wider accessibility/design blast radius.
  • Widening the entire Settings page would hide the table defect and affect every Settings route. The table must solve its own layout.
