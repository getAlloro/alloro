Local Rankings Refresh Map Hotfix

Status
Completed
Plan Folder
plans/07152026-local-rankings-refresh-map-hotfix
Size
Small
Risk
Level 2

Why

In Pilot mode, refreshing Local Rankings competitors can leave the Leaflet map visually above the confirmation overlay. The pilot bug-report action can also fail before the ticket form opens when screenshot capture fails.

What

Fix the refresh confirmation layering so the modal reliably covers the map at narrow Pilot widths, and make the support launcher open the bug-ticket composer even when screenshot capture is unavailable.

Context

Relevant Files

  • frontend/src/pages/competitor-onboarding/LocationCompetitorOnboarding/RefreshSuggestionsConfirmModal.tsx - refresh confirmation overlay shown from Local Rankings competitor management.
  • frontend/src/pages/competitor-onboarding/LocationCompetitorOnboarding/CompetitorMap.tsx - Leaflet map whose panes use high z-indexes.
  • frontend/src/components/support/GlobalSupportAction.tsx - global bug/feature/support launcher and screenshot capture handoff.
  • frontend/src/contexts/SupportQuickActionContext.tsx - pending support draft type shared with the Help route.
  • frontend/src/pages/Help.tsx - existing route already tolerates missing attachment files at runtime.

Patterns To Follow

  • Code Constitution §12.1 and §14.1: keep frontend behavior in existing feature components/context, with no new API path.
  • Code Constitution §16.2 and §16.3: do not swallow support capture failures; surface a toast and continue the ticket flow.
  • Code Constitution §17.1 and §17.2: no console output and no any.

Reference file: frontend/src/components/support/GlobalSupportAction.tsx - current support launcher flow to preserve.

Constraints

Must

  • Keep the hotfix branch based on origin/main.
  • Validate locally with the browser at a narrow viewport around the attached screenshot size.
  • Merge the fix forward to dev/dave before opening the PR to main.

Must Not

  • Do not add dependencies.
  • Do not change the competitor refresh API behavior.
  • Do not make screenshot capture mandatory for support ticket creation.

Out Of Scope

  • Changing competitor discovery logic, ranking rerun behavior, support ticket backend behavior, or production deployment.
  • Alloro Docs updates; this is a behavior-only hotfix with no new guidance, labels, or user-facing documentation.

Risk

Level: 2 - UI layering and fallback behavior can regress if z-indexes fight other global overlays or if support drafts assume a screenshot.

Risks Identified

  • Raising the refresh modal too high could cover the Pilot mode badge. Mitigation: set it above Leaflet but below the existing z-[9999] Pilot badge.
  • Making screenshots optional could leave users without visual evidence. Mitigation: still attach console logs, preserve source URL, and tell the user the ticket opened without a screenshot.

Blast radius: Local Rankings competitor-management confirmation modal; global support launcher; Help ticket composer draft handoff.

Pushback

  • This should not be fixed by shrinking the map or changing ranking layout. The root bug is stacking order: Leaflet panes are above a low-z modal.
