Why

The four hub redesigns shipped page by page; this pass makes them read as ONE product per the three-pillar brief: white cards on linen, ink-black text, terracotta as the only accent, the same card recipe and status language everywhere — plus the data-correctness fixes (YTD definition, dead deltas, mislabels) that erode trust in the numbers.

Owner decisions locked: fonts stay as-is (Spectral display + Jakarta body — "one font everywhere" is OFF); numbers are ink by default with terracotta = attention; reveal-next is the lightweight collapse (no backend); Rankings M/Q/YTD toggle stays dormant.

What

A client-surface sweep (Practice Hub, Referrals Hub, Local Rankings, Reviews & Posts + shared nav) plus targeted data fixes. Done when:

  • Global background is linen #F7F3ED (sampled from the approved friyay boards; current --color-alloro-bg is cool-gray #F3F4F6); cards stay white.
  • Primary copy on the four hubs is ink (alloro-navy #11151C), with ONE muted tone for secondary text — no more slate-500/gray-500 mixing.
  • Stat values render ink in the same treatment family as today; terracotta (alloro-orange #D66853) marks ONLY attention states. Hardcoded off-palette hexes (e.g. #B3503E warn in GbpManagerPage StatBox) replaced by tokens.
  • One card recipe — rounded-[14px] border border-line-soft bg-white shadow-premium — across hub surfaces AND the gbp-automation/PMS components that currently use border-slate-200/rounded-2xl border-gray-200.
  • Status dots: shared tone module (green #4F8A5B healthy · amber #C2891E needs-attention · red added for overdue/failing), same semantics on every hub. statusRules.ts TONE_COLOR promoted to the shared source of truth.
  • To-Do List and Notifications hidden from Sidebar (Execution group removed if emptied) and MobileBottomNav; the 3-second notification poll removed with them; routes remain reachable by URL.
  • Action banners (Practice Hub OneThingBanner + Referrals + Rankings 1-ACTION) get "Mark done" → collapses to "Next action arrives with your next analysis." — localStorage keyed, no backend.
  • Referrals top-sources rows are a click-in: inline expand showing the per-month trend (data already computed in sourceTrend.ts).
  • YTD honesty: ProductionPanel no longer sums ALL months under a YTD label when no current-year rows exist (falls to an honest empty/zero state); duplicate-month double-counting in usePmsKeyData ruled out or fixed (the "$2.2M reads wrong" suspect).
  • "New patient starts" mislabel: confirmed dead (lives only in unrendered Trajectory.tsx) — guarded by the dead-file note below.
  • +0% deltas and "phases/sections/prior period" labels: investigated on the live surfaces, fixed where found, or reported back with exact locations if they live in flows needing owner input.
  • Mobile spot-check: 4 hubs render correctly at 375px (no horizontal scroll, grids collapse).
  • tsc (both) + eslint clean.

Blast radius: index.css token (global); statusRules.ts gains consumers (3 hubs); Sidebar/MobileBottomNav (nav-wide); ProductionPanel/usePmsKeyData (Practice Hub numbers); no backend.

  • "Solid black" implemented as ink #11151C, not #000 — it IS the brand black; pure black on linen is harsher.
  • All-terracotta numbers rejected (owner-confirmed): terracotta must keep meaning "look here".
