⛔ NO EMAILS — quarantined flows (DO NOT TRIGGER)

This system sends real email in all environments. Do not perform any of these during testing:

  • Login: do NOT use OTP login (emails a code), password reset / forgot password, resend verification, or create a new signup (welcome/checkup emails). Log in with an existing password session only.
  • Organizations → Set User Password with the "notify user" option ON → emails a temp password. If you must test it, turn notify OFF; otherwise SKIP.
  • Agent Runs → monthly-agents-run (LIVE) → sends an admin summary email (+ creates tasks + fires external webhooks). Use monthly-agents-run-test instead (read-only, no DB writes, no email).
  • PMS → Upload-with-mapping → Approve → kicks off the monthly-agents pipeline (→ email). Test the mapping preview only; do NOT complete the upload→approve→automation chain.
  • Public Contact Form submission (on a rendered site) → emails the practice's recipients. Do not submit a notifying form.
  • Support → reply to a ticket → may email the requester. View tickets only; do not send replies.
  • Referral conversion flows → reward/notification emails. Do not convert referrals.
  • Email Manager / any "send test email" control → obviously skip.

Also avoid outbound-to-Google actions even though they aren't email: GBP review-reply publish and GBP local-post deploy push to live Google Business Profiles. View only.
