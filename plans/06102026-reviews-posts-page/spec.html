Why

Fourth and final dashboard redesign (Practice Hub → Referrals Hub → Local Rankings → Reviews & Posts). The Engage workspace is buried as a secondary tab inside Local Rankings; reviews are "too important to hide behind a toggle." It becomes its own sidebar page at /gbp-manager — the URL three existing arrows already point at (Practice Hub Reviews card, plus the planned route reserved since redesign #1).

What

Promote the existing GbpAutomationPanel to a standalone page with a simplified board-04 shape. Done when:

  • /gbp-manager renders a "Reviews & Posts" page in the shared 960px container: header → navy pill tabs (Reviews · Posts · Settings, Reviews default) → per-tab content.
  • Reviews tab leads with three stat boxes — NEEDS REPLY (N · last 60d), LAST REVIEW (Xd ago · keep it fresh), COVERAGE (NN% · replied) — then the existing reviews workspace scoped to a 60-day chunk with a "view older" escape into the month-bucketed view. The trend sparkline is dropped.
  • On load, 1–2 reply drafts are silently pre-generated for the newest unreplied reviews without existing work items (guarded; see T4).
  • Posts tab is the existing posts manager with the photo now optional end-to-end (composer, work-item edit, backend validation, Google deploy payload).
  • "Reviews & Posts" appears in the Sidebar (after Local Rankings) and MobileBottomNav; the Practice Hub Reviews card's /gbp-manager arrow resolves.
  • Local Rankings loses its Engage tab (Overview-only); no orphaned imports.
  • No "keywords to rank for" input exists (confirmed: none ever did). No new endpoints. tsc + lint clean.

Constraints

Must

  • Reuse GbpAutomationPanel + children — no rebuild of the mutation/dialog wiring.
  • Pre-gen guards: once per mount per location (ref), cap 2, skip reviews with existing work items, only when readiness.ready && settings.review_reply_enabled.
  • Photo-optional must omit media from the Google payload when absent (not send null/empty).
  • 60-day chunk derived client-side from review_created_at; "view older" reveals the existing month-bucketed UI.
  • Stat boxes read the SAME useGbpAutomation queryKey (no duplicate fetch).
  • Keep Settings reachable (third tab) — automation toggles + sync live there.

Must Not

  • No new endpoints; no scheduler/cron for pre-gen.
  • No changes to reply/post deploy flows beyond photo-optional.
  • Do not delete RankingsDashboardViewTabs or GbpEngagementSummaryCard files (legacy branch references them); stop rendering only.
  • No wizard tour additions this pass (nothing currently targets Engage; adding a tour page is separate scope).
  • No "keywords to rank for" feature.

Out Of Scope

  • Backend 60d counter (client-derived instead); review-sync changes; post scheduling changes.
  • Wizard tour page for /gbp-manager.
  • Deleting legacy rankings components (separate cleanup once the redesign is confirmed).

Risk

Level: 3 (new top-level route/nav + page-load LLM calls + a backend validation relaxation touching the Google deploy payload).

Risks Identified

  • R1 — Pre-gen runaway (Level 3). Page-load LLM calls could re-fire on refetch/StrictMode/remount. Mitigation: once-per-mount-per-location ref guard; existing-work-item skip; cap 2; readiness + review_reply_enabled gates; backend "already generating" check is the backstop. Silent failure tolerated (user can still click Draft reply).
  • R2 — Photo-optional payload (Level 2). Mitigation: conditional media in the deployment service; relax (not remove) URL sanitation; loosen the two API signatures + modal gate together so no layer still hard-requires it.
  • R3 — Panel prop surface (Level 2). Controlled view + hide flags + window prop are additive with defaults = current behavior.
  • R4 — 60d window vs month sidebar (Level 2). Window mode replaces the sidebar with a flat recent list + "view older" toggle back to the existing UI; no data changes.
  • R5 — Mobile nav fit (Level 1). Verify a 4th item fits MobileBottomNav; truncate label if needed ("Engage").

Blast radius: GbpAutomationPanel's only consumer is RankingsDashboard's engage branch (being removed) → the new page becomes the sole consumer; additive props can't break others. Photo-optional touches draft creation + deployment for posts only (review replies untouched). Nav: App.tsx/Sidebar/MobileBottomNav additions are isolated. GbpEngagementSummaryCard remains legacy-only.

Pushback

  • Pre-gen on load spends LLM tokens for any visitor with unreplied reviews. Capped at 2 and once-per-review-ever (work items persist), so worst case is bounded — but if cost telemetry later shows waste, the same code moves behind a "Generate latest" button trivially.
  • Tabs contradict the board's "not behind a toggle" letter — owner explicitly chose pill tabs (reviews default, so reviews are never hidden on landing). Noted as a deliberate decision.
