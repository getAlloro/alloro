Why

Admin OS accepts Word, Excel, PDF, and Markdown files, but the current PDF converter discards table geometry and embedded images before the Markdown renderer receives them. Wide GFM tables produced by DOCX/XLSX also lack a bounded horizontal-scroll treatment. The result is misleading: the upload succeeds and indexes, but the stored document no longer preserves the source's useful structure.

This plan is a focused fidelity follow-up to plans/07042026-alloro-os-admin-port. It does not rewrite that historical plan or expand OS into a second knowledge system.

What

Upgrade the existing import pipeline so supported formats preserve their semantic content in the Markdown system: headings, lists, basic tables, and renderable raster images. PDF conversion uses deterministic extraction first and a bounded Gemini page-layout fallback only when a page is low-text, table-heavy, or multi-column. Extracted images remain real assets stored through OsAssetService; Gemini never fabricates image URLs.

Definition of correctly rendered: semantic fidelity and usable reading, not pixel-perfect cloning of an arbitrary office document. The original file remains the exact source artifact in S3.

Constraints

Must

  • Use the existing pdf-parse, pdfjs-dist, JSZip, SheetJS, Gemini, S3, and Markdown stack. No new npm dependency without a spec revision (§4.4).
  • Keep converter orchestration split into single-purpose files rather than expanding pdfConverter.ts into a large parser (§2.1-§2.4).
  • Run deterministic extraction first. Vision is a bounded fallback and must degrade to readable text plus warnings when Gemini is missing, busy, or malformed (§3.1-§3.3).
  • Send page images to Gemini only through service.os-llm.ts; never log image bytes, base64, document text, credentials, or presigned URLs (§5.3, §9.3).
  • Persist extracted images through OsAssetService. Converters return placeholders and buffers; they do not write S3 or the DB directly (§7.1).
  • Use named, startup-validated page width/page count/image threshold limits in osKnowledgeBase.ts (§4.2, §5.6).
  • Keep the import job idempotent and preserve existing retry/failure behavior (§21.1-§21.4).
  • Give both read and edit tables bounded horizontal scrolling, visible keyboard focus, and preserved native table semantics.
  • Use synthetic committed fixtures only; the two supplied internal PDFs are manual acceptance inputs and must not enter Git (§20.4).

Must Not

  • Do not promise pixel-perfect reproduction of arbitrary PDF/DOCX/XLSX layout.
  • Do not add a second asset store, direct public S3 URLs, raw HTML in Markdown, or data URLs in stored content.
  • Do not let Gemini invent source text, rows, image URLs, citations, or missing content. Malformed output falls back and warns.
  • Do not silently discard unsupported images or truncated layout work; record a specific import warning.
  • Do not change OS routes, schema, RAG indexing, chunking, version semantics, or client-facing dashboard surfaces. Rev 5 may delegate only the read-only OS asset prefix through the app-level guard to its existing route-specific JWT and super-admin middleware; do not broaden or weaken any other auth boundary.
  • Do not touch unrelated changes in alloro-docs or the two remote commits currently ahead of the local branch.

Out Of Scope

  • Pixel-perfect rendering, page-layout editing, charts as live data visualizations, macros, embedded video/audio, SVG/EMF/WMF execution, or sidecar-folder imports.
  • Embedded-image extraction from legacy binary .xls. The converter must preserve cell tables and warn honestly.
  • Reprocessing old imported documents automatically. Existing documents can be re-uploaded after the fix; a batch backfill needs a separate plan.
  • Alloro Docs parity changes. OS is internal super-admin-only and has no client docs entry; this check is recorded rather than skipped.

Risk

Level: 2 - Concern. The change touches the conversion worker, sends selected page screenshots to the existing Gemini provider, writes extracted images to the existing AWS S3 bucket, and changes reader/editor presentation. It is additive and has no schema or auth change, but malformed AI output or unbounded page processing could corrupt content or amplify cost.

Risks Identified

  • AI transcription can omit or invent structure. Mitigation: deterministic extraction first; strict exact-transcription prompt; image-marker contract; Markdown validation; bounded retry; fallback to deterministic text and a persisted warning.
  • Latency and Gemini cost can scale with page count. Mitigation: invoke vision only for low-text/table/layout/image-placement pages; render at a named width; process sequentially or with very small bounded concurrency; cap vision pages with an env-overridable limit and warn when exceeded.
  • PDF images lack reliable MIME/position metadata. Mitigation: sniff PNG/JPEG/GIF/WebP signatures, reject unknown formats, use page-order markers when vision succeeds, and append unmatched extracted images to the correct page section with a warning.
  • XLSX OOXML relationships vary by producer. Mitigation: support standard worksheet/drawing/media relationships, sort by anchor row/column, ignore unsafe paths, and fall back to table-only with a warning for unsupported packages.
  • Live acceptance writes external state. Mitigation: automated tests mock S3/Gemini; the testing phase explicitly names the AWS/Gemini blast radius, uses test-prefixed documents, and removes documents, assets, and S3 objects after evidence is captured.
  • Wide-table wrappers can break semantics or editor layout. Mitigation: keep the real <table>, add a focusable overflow region only around it, target TipTap's existing .tableWrapper, and test keyboard/overflow behavior.
  • The app-level default-deny guard currently intercepts image requests before the OS asset route can read its query JWT. Mitigation: delegate only /api/admin/os/assets/* to the existing route-specific JWT verifier and super-admin middleware; test missing and invalid tokens and keep every other Admin OS path on header auth.
  • Editor parity changes touch ProseMirror selection, deletion, and Markdown serialization. Mitigation: port only the final proven Alloro OS Rev 8/15/17-20 design, keep the plain table node (not the failed table node-view attempts), isolate image/delete logic in dedicated extensions and node views, and add model/round-trip/component regression tests.

Blast radius: Admin OS import conversion, OS Gemini provider interface/fakes, OS config parsing/tests, OS asset placeholder and delivery flow, the app-level default-deny allowlist for one read-only path, the shared OS shell width, OS Library drop behavior, OS reader/editor tables, upload-format copy, and conversion/integration tests. No database table, route definition, RAG algorithm, version model, or client dashboard consumer changes.

Pushback

Do not define success as lossless conversion. Markdown cannot represent every merged cell, floating object, chart, macro, or PDF drawing operation. The recommended contract is semantic fidelity plus visible warnings and retention of the original source object. Future exact visual preview belongs in the adjacent original-file preview feature.
