Why

Agents need a portable command contract for isolated worktree acceptance without port collisions, repeated manual authentication, production email, or unintended database writes. Planning also needs to stop contaminating the primary checkout by creating a linked worktree before every structured plan by default.

What

Add -tw/--test-worktree to the shared Claude and Codex instructions, make -s create a secondary linked worktree from the current branch HEAD by default, and document Alloro-specific paths and safety expectations. This change defines the command contract only; a separate implementation is still required before Alloro can actually run npm run test:worktree.

Context

  • ~/.claude/CLAUDE.md — authoritative shared command workflow.
  • ~/.codex/AGENTS.md — Codex mirror of the shared workflow.
  • alloro-chat/AGENTS.md — the only other active project file that is an exact shared-workflow mirror.
  • alloro/CLAUDE.md and alloro/AGENTS.md — Alloro entrypoint and project-specific operating rules.

Reference pattern: the existing synchronized command table and mode sections in ~/.claude/CLAUDE.md.

Constraints

Must

  • -tw hard-fails outside a secondary linked worktree.
  • -s creates a new secondary worktree from the current branch HEAD unless the user explicitly passes --no-worktree.
  • Use OS-assigned ports and isolated DB, auth, mail, queue, worker, and external-write policies.
  • Keep the three shared instruction copies byte-identical.

Must Not

  • Do not claim that the Alloro runtime adapter exists.
  • Do not copy uncommitted source-checkout changes into a new planning worktree.
  • Do not overwrite custom instructions in other project repos.
  • Do not mutate unrelated Alloro application work.

Out Of Scope

  • Implementing the portable runner, containers, database seeding, browser harness, or Alloro test:worktree package script.

Risk

Level: 2 — command behavior changes globally and can create branches and sibling worktree directories.

  • Agents could mistake policy for an implemented runner. Mitigation: require a repository adapter and fail safely when it is missing.
  • Uncommitted source work could be assumed present in the new worktree. Mitigation: explicitly report that only the current commit is used and never copy or stash changes silently.
  • Custom project rules could be lost through broad synchronization. Mitigation: sync only exact global mirrors and add narrow Alloro-local sections.

Blast radius: future Claude/Codex command handling globally, Alloro sessions, and Alloro Chat sessions using its exact mirror.
