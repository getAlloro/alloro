Why

Alloro currently loads and directly references Fraunces-derived display serif typography. The hotfix replaces that serif identity with Spectral so production and dev render the requested brand font consistently.

What

Replace live display-serif references with Spectral in the frontend font loader, the global display token, direct dashboard card font stacks, and the rankings SVG font stack. Do not rewrite historical plans, changelog entries, generated snapshots, or unrelated serif/sans email fallback stacks.

Docs impact: no Alloro Docs update is required because the hotfix changes only the display font family, not dashboard copy, controls, tooltips, navigation, permissions, or user guidance.

Context

Relevant Files

  • frontend/index.html - Google Fonts loader for the SPA.
  • frontend/src/index.css - global --font-display token consumed by .font-display.
  • frontend/src/components/dashboard/focus/PMSCard.tsx - direct display serif constant.
  • frontend/src/components/dashboard/focus/LocalRankingCard.tsx - direct display serif constant.
  • frontend/src/components/dashboard/focus/WebsiteCard.tsx - direct display serif constant.
  • frontend/src/components/dashboard/RankingsDashboard.tsx - SVG text font stack.
  • frontend/src/components/dashboard/shared/SectionTitle.tsx - live comment naming the display font.

Patterns To Follow

  • Keep the established font-display utility and update the token behind it rather than touching every consumer.
  • Preserve existing fallback behavior with Literata, Georgia, and generic serif after Spectral.

Reference file: frontend/src/index.css - existing token-based typography structure.

Risk

Level: 2 - Concern

Risks Identified

  • Direct hardcoded display-serif stacks bypass the global token. Mitigation: replace direct hardcodes and verify no live Playfair Display or Fraunces references remain.
  • Font loader mismatch can silently fall back to another serif. Mitigation: update the Google Fonts URL and build the frontend.

Blast radius: authenticated dashboard display headings, focus dashboard product cards, rankings gauge SVG label, and login/signup display headings through font-display.

Pushback

  • This should stay a token swap, not a component sweep. Touching every font-display consumer would create noise and future-us would hate the pointless diff.
