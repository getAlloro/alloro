Remove AI Data Insights

Why

The admin AI Data Insights / Agent Enhancements surface is a retired Guardian/Governance review workflow. The run route is already disabled, the page mostly shows an empty state, and keeping the view plus dead job code creates a misleading admin path.

What

Remove the admin navigation item, list/detail routes, frontend view code, /api/admin/agent-insights API surface, and dead Guardian/Governance runner code. Direct visits to old /admin/ai-data-insights URLs should redirect to Mission Control instead of rendering the retired page or an empty admin shell.

Context

Scope Ledger

  • Objective: remove AI Data Insights from admin, including view access and dead Guardian/Governance job code.
  • Intended user: internal Alloro admin users.
  • Surface: admin SPA routes/sidebar, admin API route, legacy agent runner service.
  • System of record: code only. No schema or data deletion is planned.
  • Open questions: none blocking. Direct stale route behavior is specified as redirect to Mission Control.

Relevant Files

  • frontend/src/pages/Admin.tsx - owns admin route registration for ai-data-insights and ai-data-insights/:agentType.
  • frontend/src/components/Admin/shell/AdminSidebar.tsx - owns the Agent Enhancements nav item.
  • frontend/src/pages/admin/AIDataInsightsList.tsx and frontend/src/pages/admin/AIDataInsightsDetail.tsx - retired list/detail views to delete.
  • frontend/src/hooks/queries/useAdminStandaloneQueries.ts, frontend/src/lib/queryClient.ts, frontend/src/types/agentInsights.ts - frontend query/type surfaces used only by this retired view.
  • src/app.ts and src/routes/adminAgentInsights.ts - backend route mount and router for /api/admin/agent-insights.
  • src/controllers/admin-agent-insights/ - backend controller/services/utils dedicated to the retired API.
  • src/controllers/agents/AgentsController.ts and src/routes/agentsV2.ts - contain stale Guardian/Governance endpoint docs, disabled route references, dead controller export, and health fields.
  • src/controllers/agents/feature-services/service.governance-validator.ts, service.recommendation-parser.ts, and service.agent-input-builder.ts - dead Guardian/Governance run and payload parsing path.
  • src/models/AgentRecommendationModel.ts and src/models/AgentResultModel.ts - contain helper methods used only by the retired admin insights API and dead runner; keep shared model methods used by reset and Agent Outputs.

Patterns To Follow

  • §6.1 backend reference: keep route/controller/service/model layering clean by removing entire unused domain files instead of leaving dangling exports.
  • §12.1 frontend reference: the removed frontend query code currently follows the API + React Query + type triad; remove all three together so no dead query keys remain.
  • §4.1 no commented-out code: the older reversible disable comments in agentsV2.ts should be deleted with the dead endpoint docs.

Reference files: src/routes/admin/agentOutputs.ts and src/controllers/admin-agent-outputs/ stay as the current supported admin agent-output surface; frontend/src/pages/admin/AgentOutputsList.tsx stays as the current admin agent output view.

Constraints

Must

  • Remove the sidebar item and both AI Data Insights routes from admin.
  • Redirect stale /admin/ai-data-insights/* URLs to /admin/mission-control.
  • Remove the backend /api/admin/agent-insights route mount and dedicated controller domain.
  • Remove dead Guardian/Governance runner code and stale disabled route comments.
  • Preserve shared agent_results and agent_recommendations tables and model methods used by live reset/output flows.
  • Re-check Alloro Docs references during execution; current grep found no matching docs references.

Must Not

  • No database schema changes or data deletes.
  • No changes to Proofline, Ranking, monthly agents, Copy Companion, Agent Outputs, PMS reset, or org reset behavior.
  • No removal of shared callAgentWebhook; it is still used by monthly test and Copy Companion code.
  • No broad admin shell refactor or unrelated navigation cleanup.

Constitution Articles

  • §4.1 delete commented-out dead code rather than preserving old disabled blocks.
  • §7.1, §7.2, §7.3, §7.4 preserve backend route/controller/service/model boundaries while removing a whole unused domain.
  • §8.1 unmatched API calls should fall through to the existing standard 404 envelope after auth.
  • §11.1 protected API behavior remains guarded by default-deny auth before the 404 fallback.
  • §12.1, §14.1, §14.2 remove the frontend query triad and avoid leaving raw or orphaned API callers.
  • §20.5 execution must generate acceptance artifacts before finalization.

Out Of Scope

  • Cleaning historical database rows for guardian or governance_sentinel.
  • Removing guardian / governance_sentinel display support from the separate Agent Outputs list.
  • Removing Copy Companion or GBP optimizer code beyond stale Guardian/Governance references.
  • Production deployment or live DB verification.

Risk

Level: 2 - concern. The code path is retired, but the cleanup crosses frontend, backend, and legacy agent files.

Risks Identified

  • Over-deleting shared agent infrastructure. agent_results and some AgentRecommendationModel helpers are still live for Agent Outputs and org reset. Mitigation: remove only methods with no remaining references after deleting the retired domain; rerun targeted grep before and after edits.
  • Stale direct URLs. Removing routes alone can leave an empty admin content area. Mitigation: add an explicit stale-route redirect to mission-control.
  • Hidden internal callers. A script or stale test might still hit /api/admin/agent-insights or guardian-governance-agents-run. Mitigation: grep full repo during execution and update tests or docs if any current caller remains.
  • Docs drift. Admin navigation removal can affect docs. Mitigation: grep /Users/rustinedave/Desktop/alloro-docs again during execution; update only if references exist.

Blast radius: Admin shell navigation, admin SPA route table, admin standalone query hooks, /api/admin/agent-insights, stale Guardian/Governance runner exports, related model helper methods, and any tests that explicitly expected that retired endpoint.

Pushback

  • Do not delete the database tables. The table names look tied to this feature, but current reset and output flows still depend on them. Future-us will hate a cleanup that turns into a data loss bug.
  • Do not leave disabled comments in place. The prior plan intentionally kept code for restoration; this request is removal, so dead comments now create architectural drift.
