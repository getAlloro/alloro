Why

A2 (the completeness monitor, plans/07152026-gbp-own-completeness-scoring) already detects that a Google Business Profile field is empty, and A6 (the write-back, plans/07152026-gbp-writeback) can push an owner-approved change to Google — but nothing connects them. The "found it → fixed it" arc is broken at its first seam. This closes it: a detected, fillable gap becomes an owner-approval draft to fix the profile. The primary category is the strongest get-found lever, so closing this arc is where get-found improvements start.

What

An owner/operator-triggered endpoint — POST /api/gbp-automation/business-info/completeness-fill — that grades a location's GBP completeness live and stages a draft businessInformation write-back for the gaps Alloro can fill from a value it genuinely holds independent of Google. Nothing auto-publishes: the owner approves before any write, the write-back master switch is re-enforced, and the draft flows through A6's existing approval/deploy path.

Done when: the endpoint stages an owner-approval draft (201) for a fillable gap and returns an honest empty (200) when nothing is fillable, verified behaviorally on real data.

Constraints

Must

  • Never stage a fabricated or blank value (Value #6). Today only websiteUri has a trustworthy independent source (the practice's own domain).
  • Re-enforce the write-back master switch and Google-readiness before staging.
  • Tenant scope from server context (§5.5/§11.7); auth + validation at the boundary (§11.1/§11.2).
  • Response contract { success, data, error } via the domain builders (§8.1/§8.2).

Must Not

  • Auto-publish. Nothing reaches Google without owner approval downstream.
  • Guess category / phone / hours — surfaced as no-value-source, never filled (they stay owner-verified).
  • Touch unrelated GBP flows.

Out of scope

  • The value-source for category/phone/hours (a separate owner-input build). The completeness → owner-surface trigger (its own follow-on PR).

Constitution Articles touched: §6.1, §7.4, §8.1/§8.2, §5.5/§11.7, §11.1/§11.2.

Risk

Level: 2 — a write-back path to a customer's live Google profile, but gated: draft-only, owner-approved, master-switch-enforced, and A6 ships disabled per-account. No auto-send.

  • Over-fill / fabricated value. Mitigation: only fill a field with a genuine independent source; everything else is no-value-source, surfaced not guessed.
  • Circular source. Mitigation: business_data is populated FROM the GBP read, so it is never used to refill a GBP gap.

Blast radius: additive endpoint + service in the gbp-automation domain; no existing field changes. Downstream draft/approval/deploy path unchanged.
