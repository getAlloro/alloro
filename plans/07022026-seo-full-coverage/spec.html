Why

Follow-up to plans/07022026-seo-metatag-fixes (Completed). A fresh, precise audit of all 57 pages and 254 posts across the 4 sites (One Endodontics, Artful Orthodontics, Garrison Orthodontics, getalloro.com) found that the prior fix only ever touched posts — pages still carry the same invalid-schema-type, missing-rating, and missing-FAQ-schema defects untouched. It also surfaced something new and bigger: 92 One Endodontics posts and 16 Garrison posts have no SEO data at all (no title, no description, nothing was ever generated), and Garrison's other 33 posts have a title but zero schema_json. Google is auto-writing titles/snippets for 108 live, published pages today.

Close every gap that is honestly closeable without fabricating data:

Constraints

Must

  • Read-patch-write every page/post's seo_data — never assume a full-column write is additive (same rule as the prior plan; applies equally to PageModel's write path).
  • Title trim must preserve the front keyword+location segment; only drop trailing segments (typically the brand-name tail).
  • T3/T4 generation must run through the existing tiered pipeline (already prompt-fixed for schema type) — no new prompt content authored in this plan.
  • Test the title-trim function and the page-enrichment path against one real page/post before batch-applying either.
  • Test T3's generation against one real blank post before running it against all 108.

Must Not

  • Do not fabricate faq_candidates or a rating for sites/pages that have no real underlying data (Artful/Garrison/getalloro.com pages' FAQ gap, getalloro.com's rating gap) — leave those cells as they honestly are.
  • Do not re-run generation on any post/page that already has a meta_title — T3/T4 target only rows confirmed to have zero seo_data or zero schema_json, by id, not a blanket re-generate.
  • Do not touch getalloro.com posts (still none) or the two archived ghost projects — same exclusions as the prior plan.

Out Of Scope

  • Wiring the geo_layer section to actually produce faq_candidates for Artful/Garrison/getalloro.com content that currently has none — a real content-generation feature, not a data-repair task.
  • The archived-project-still-renders bug and the renderer soft-404/sitemap gaps — already tracked separately (background task task_99606846).

Risk

Level: 3 — real LLM generation against live, published, publicly-served content for the first time this session (the prior plan was enrichment-only, zero LLM calls in its backfill).

Risks Identified

  • Level 2 LLM generation for ~141 posts (108 zero-data + 33 schema-only) has real API cost and takes real time (sequential, per the existing bulk pattern) — this is not free or instant like the prior enrichment-only pass. Mitigation: test on one post first; the tiered runner (3 rounds, not 6) already minimizes call count.
  • Level 1 Freshly-generated content for 108 currently-blank posts is new, unreviewed AI copy going live automatically. Mitigation: regression risk is near-zero (nothing currently shows to users/Google for these posts, so any real content is an improvement) — proceeding without a per-post human review gate, matching the existing bulk-generate precedent. Flagged for the owner to override if a review pass is wanted instead.
  • Level 2 The title-trim function operates on live, already-decent titles — an aggressive or naive trim could cut something important. Mitigation: only drop whole trailing pipe-delimited segments (never mid-word truncation), verified against a handful of real over-length titles before batch-applying.
  • Level 1 Same full-column-replace write-path risk as before, now also applying to PageModel. Mitigation: identical read-patch-write discipline as the prior plan's enrichPostSeoData, applied to the new page-level function.

Blast radius: Same shared generation engine as the prior plan (service.seo-generation.ts, util.seo-section-runner.ts) — platform-wide by nature. Directly written: website_builder.pages.seo_data for up to 57 rows (4 sites), website_builder.posts.seo_data for up to 141 rows (2 sites, generation) plus title-trim touching any over-60-char row on top of the prior plan's already-enriched 254 posts.

Pushback

  • Not every red/yellow cell in the audit table is a defect — see What. Forcing FAQ schema onto content with no real Q&A material, or a rating onto a SaaS product with no public reviews, would be the same kind of fabrication this whole effort exists to remove. Recommending those stay as-is rather than "solving" them cosmetically.
