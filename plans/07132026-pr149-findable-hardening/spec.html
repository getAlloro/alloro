Why

PR 149 compiles and passes the existing suites, but direct contract tests exposed four owner-facing ranking-band contradictions. The PR must preserve measured search-position truth before it can be merged into dev/dave.

What

Correct the #1, #2-3, #4-20, not-in-top-20, and unavailable bands across the dashboard helper, LLM prompt, and output sanitizer. Add deterministic tests that fail on the four reviewed regressions.

Context

  • frontend/src/components/dashboard/rankingsDashboard.utils.ts - deterministic owner-facing fallback copy.
  • src/controllers/practice-ranking/feature-services/service.ranking-llm.ts - ranking analysis prompt contract.
  • src/controllers/practice-ranking/feature-services/service.ranking-output-guardrails.ts - deterministic post-LLM normalization.

Patterns: Constitution §6.1 for backend domain placement, §12.1 for frontend boundaries, and §20.1-§20.2 for mirrored contract tests. Frontend test analog: frontend/src/components/PageEditor/seoPanel.utils.test.ts.

Constraints

Must

  • Use searchPosition and searchStatus only.
  • Keep all new logic pure and deterministic.
  • Preserve existing rank calculations and persistence.

Must Not

  • No migrations, routes, dependencies, live LLM calls, or database access.
  • No unrelated prompt or dashboard redesign.
  • No production or dev deployment in this task.

Docs impact: No Alloro Docs change. This corrects dynamic generated/fallback wording without changing controls, labels, navigation, screenshots, or user guidance.

Risk

Level 2: Shared ranking copy can affect every newly generated ranking analysis and the next monthly summary that consumes its recommendation.

  • Prompt drift. Mitigation: encode mutually exclusive bands and add a prompt-contract test.
  • Frontend/backend wording drift. Mitigation: test the dashboard helper and sanitizer with the same #1/#2/#4/not-top-20 cases.

Blast radius: Local Rankings overview copy, persisted LLM ranking analysis text, and downstream Summary consumption of ranking recommendations. Ranking scores, Maps positions, competitor selection, and database rows are unchanged.
