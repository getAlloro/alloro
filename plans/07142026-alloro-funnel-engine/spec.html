Why

Alloro's job is to increase the number of opportunities to convert — not the practice's internal conversion of a lead they already have (that is their operations, out of lane). The way Alloro adds opportunities is the funnel: get found → get considered → get chosen.

Search is going zero-click into Google's own surfaces and AI answers, but the same underlying signals — complete structured data, a consistent entity, answer-first content — feed both the still-dominant local map pack and the emerging AI-answer surface. Alloro hosts the practice's site and manages its GBP, so it owns those surfaces and can hold them to the standard directly.

Slice 1 lays the foundation by extending capability that already exists, not rebuilding it, and splits into two reviewable PRs: 1a read-only analysis + the honesty gate; 1b the write path + the dogfood.

Rev 5 — 2026-07-16

Change: Slice 1a third-round fix (PR #158, pinned to bfd9dc11) — the honesty gate's negation-scope boundary set, reworked as a class. Rev 4's claim that the guard is "clause-accurate" is withdrawn as an overclaim: its boundary set enumerated only some of the ways a clause can open, so a negator's scope ran past boundaries it should have stopped at, and an adversary broke it on shapes Rev 4 had not enumerated. Root cause: Rev 4 fixed the reported shapes (adversative conjunctions) rather than enumerating the boundary space. Boundaries are now grouped by what licenses a clause split. Hard: sentence/clause punctuation and the ellipsis; a paragraph break or a new list item; an adversative, conjunctive adverb, or causal subordinator; a leading subordinate clause consumed through its closing comma; and a leading verbless negated noun-phrase fragment consumed through its closing comma (a determiner-no with no verb to attach to cannot reach a following predication). Soft — a separator ends scope only when what follows opens a new predication: a comma, paren, line break or tab + a new subject; a coordinating conjunction + a new subject; a dash or block separator (title-tag pipe, bullet, arrow, slash) + anything not negation-carrying. Deliberately excluded: a bare comma and bare and/or/yet/nor with no new subject — these coordinate verb phrases sharing the negated auxiliary, where negation genuinely distributes, so splitting would over-block honest disclaimers. Three bugs were found by probing this round's own fix: a lone line break treated as hard over-blocked soft-wrapped honest prose (only a paragraph break or list marker is hard now); the dash rule's trailing whitespace could backtrack past a shared-auxiliary coordinator (the lookahead re-skips whitespace); and unbounded \s* beside a literal class was quadratic on a long whitespace run (~2 ms → ~2.6 s on a 20 KB input; bounded now, ~6 ms). Named residuals, left open on purpose because each is surface-identical to honest copy and closing it needs a POS tagger, not more regex: (1) a clausal negator + comma + a claim headed by a participle rather than a subject — indistinguishable from an honest coordinated object list, which must pass (the verbless no-fragment version of that shape is caught); (2) a comma + an open-class (proper/common noun) subject — admitting nouns would split honest object lists. Pre-existing and untouched: the phrase inventory itself over-blocks some honest copy (e.g. pain[- ]?free on a non-clinical use).

Reason: User QA: an adversary ran the live service and got safe for promise shapes Rev 4's boundary set did not cover, and the PR body's "clause-accurate" claim was falsified by those probes.

Updated Done criteria: (a) the negation-scope boundary set is enumerated by clause-opening class — punctuation, paragraph/list breaks, connectives, separators + a new subject, and dash/block separators — not by the individual reported shapes; (b) honest distributed negation still passes: a shared negated auxiliary across a bare comma or bare and/or/yet/nor, coordinated object lists under one determiner-no, parenthetical inserts, and soft-wrapped prose; (c) regression fixtures cover both directions (126 assertions) and are proven non-vacuous — 53 fail against the pre-fix service with the fixtures kept; (d) the gate's residuals are named in the module, the spec, and the PR body, and no artifact claims exhaustiveness.

Change: Slice 1a fourth-round fix (PR #158, pinned to c56797e9) — the two remaining classifier bypass classes, each fixed as a class rather than as the reported strings. (1) Boundary/encoding. Root cause named precisely: there was no normalization before gating. Every pattern in the gate is ASCII, so copy a reader sees as "guarantee" defeated it whenever it was not literally ASCII — 。/；/<br> not ending a negator's scope was one symptom of that single cause, not a boundary-list gap. A new copyNormalization.ts folds copy to what the reader sees BEFORE any match: HTML (script/style/comments removed; a break tag becomes a paragraph break; any other tag is removed without spacing, since spacing would split a word and MISS a claim; an unknown tag is treated as inline, never as a break, because a false break could cut a negator off its claim and over-block), a single-pass entity decode, removal of the invisible-formatting category (\p{Cf} + variation selectors — matched by property, not by an enumerated list of four remembered code points), NFKC (which folds fullwidth, mathematical, circled and ligature letter forms plus ；/！/？ for free), Cyrillic/Greek confusable folding, curly-quote folding, and non-ASCII sentence terminators via the Unicode Sentence_Terminal property — so 。, ।, ؟ and the rest are covered as a class, while the comma forms that property excludes stay non-boundaries exactly as the gate intends.
