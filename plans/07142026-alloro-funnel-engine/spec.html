Why

Alloro's job is to increase the number of opportunities to convert — not the practice's internal conversion of a lead they already have (that is their operations, out of lane). The way Alloro adds opportunities is the funnel: get found → get considered → get chosen.

Search is going zero-click into Google's own surfaces and AI answers, but the same underlying signals — complete structured data, a consistent entity, answer-first content — feed both the still-dominant local map pack and the emerging AI-answer surface. Alloro hosts the practice's site and manages its GBP, so it owns those surfaces and can hold them to the standard directly.

Slice 1 lays the foundation by extending capability that already exists, not rebuilding it, and splits into two reviewable PRs: 1a read-only analysis + the honesty gate; 1b the write path + the dogfood.

What — in scope

Reuse first (reconciliation — the rule this spec kept violating). Two existing subsystems get extended, not rebuilt:

  • Schema/entity: src/services/ai-seo-audit/ — identityExtractionService parses application/ld+json (incl. @graph), extracts the business entity, and discovers sameAs; entityConsistencyService, entitySignalScoring, sourceReadinessScoring present — plus stored per-page seo_data.schema_json and mature LocalBusiness/Service/FAQPage builders. Precise reuse: the parse/extraction is reused; the today's scorers check schema-type presence only, so field-level completeness grading is new logic layered on the existing extraction — not a new scorer, not an existing one either.
  • Honesty gate: GbpContentSafetyService (bounded regexes → safe/needs_review/blocked), contentPatternService (already carries the rank/visibility vocabulary: "rank #1", "first page of google", "top of google", "guaranteed ranking", "boost your ranking"), and sourceReadinessScoring.hasRiskyClaim. The lint extends these with the few missing bounded phrases.

Constraints

Honesty (Value #6) — an enforceable gate that EXTENDS existing scanners. Add bounded rank/placement/visibility/"freshness"/"will rank" phrase patterns to GbpContentSafetyService/contentPatternService; reject offending output, failing the recommendation. Every output improves eligibility/structure/trust only. Facts owner-sourced via FactExtraction; no fabricated or borrowed proof.

GBP posts guardrail. Per Google's docs, posts are customer communication to help someone decide to visit; Google names only relevance/distance/prominence as ranking factors. Posts are get-considered/engagement — never a rank lever, and no "freshness" claim.

  • Owner-approval preserved: a human approves each recommendation before it executes; publish auto-follows (existing behavior); no new autonomy.
  • GBP is read-only in Slice 1. No category/schema write; therefore no GBP rollback is in scope (it belongs to the future GBP-write slice).
  • "AEO-incomplete" is an internal/admin recommendation signal only — never owner-facing, and it gates nothing.
  • Schema truthfulness: a page may only declare a service the practice genuinely offers; the checker scores real fields, never invents one to pass.
  • Edge cases: no GBP → skip the consistency flag, score page schema from FactExtraction only; unverified/suspended listing → read/flag only (already read-only); multi-location → one page maps to one listing, sameAs/areaServed per location.
  • PII: Slice 1 uses business/owner-approved data only — no patient PII (Slice 3 sets the PII pattern).
  • Docs parity: if 1b's approval-queue UI changes, add an alloro-docs parity item; else state why none is needed.
  • Sizing (Dave): 1a and 1b are separate reviewable PRs; each is extension + wiring of existing features.
  • Lane: Corey+Claude author; Dave verifies/implements/owns branches. Route: Jo's board → Dave → alloro-engineer → PR to dev/dave.

  • Reconcile-first (the recurring failure this spec keeps making). Both the completeness score and the honesty lint were specced from scratch in earlier revs while analogs existed. Mitigation, now explicit: extend ai-seo-audit and the content-safety scanners; a reviewer should reject any new-from-scratch scorer/lint.

  • False schema. Mitigation: schema sourced only from owner-approved facts; the honesty gate rejects un-owned claims.

Change: Slice 1a second-round review fixes (PR #158, pinned to 556bd7a0). (1) Honesty-gate negation scoping — the gate's negation guard is now clause-accurate: a negator applies only to the clause it actually governs. Scope ends at sentence punctuation, at an adversative conjunction (but/however/though/…), at the comma closing a leading subordinate clause ("While X, Y" — the conjunction sits before the negator, so the closing comma is the boundary), and at a coordinating conjunction introducing a new subject. The scoping is grammatical in both directions — negation that genuinely distributes across a shared negated auxiliary ("we will not rank you #1 or get you to page one") still passes, as does "yet" used as an adverb ("we have not yet guaranteed a higher ranking"), so honest disclaimers are not over-blocked.
