Why

PR 155 removes static owner-facing claims that Google Business Profile posts improve local rank, but its ranking LLM and backend output guardrail can still generate the same false promise. The dashboard walkthrough also promises a post-due indicator that the PR removed from the Local Visibility card.

What

Make the ranking prompt and central output guardrail enforce one product rule: Google posts may be recommended for profile freshness and patient consideration, but never as a way to protect, improve, or recover rank. Preserve valid rank facts, cover both top_recommendations and overview_card.text with regression tests, and make the onboarding walkthrough describe only the rank that the dashboard actually shows.

  • R3 — prompt drift reintroduces the claim. Mitigation: state the rule in general ranking guidance, overview-card guidance, and the single-recommendation contract while keeping the deterministic guardrail authoritative.
  • R4 — docs remain misleading. Mitigation: report a separate docs parity change for the stale Local Rankings replica and Reviews & Posts guidance; do not mix it into this dirty docs checkout.

Blast radius: every newly generated ranking analysis passing through sanitizeRankingLlmAnalysis; the Local Rankings action banner and overview consumers of those persisted fields; the Practice Hub onboarding Local Visibility tooltip. Existing stored analyses are not rewritten.

Pushback

  • Selected path: narrow central rewrite plus prompt prevention. This gives deterministic protection without deleting valid rank reporting.
  • Rejected path: frontend-only filtering. It would leave false copy persisted and available to other consumers such as Summary.
  • Rejected path: remove any recommendation mentioning posts. Posting remains a legitimate engagement action; only the rank promise is false.
