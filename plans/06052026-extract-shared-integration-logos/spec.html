Extract Shared Integration Logos

Status
Completed
Plan Folder
plans/06052026-extract-shared-integration-logos
Size
Small
Risk
Level 1

Why

The four integration brand SVGs (HubSpot, Rybbit, Clarity, Google) are defined twice with byte-identical geometry: once in ActiveIntegrationLogos.tsx (compact badge variants) and again in IntegrationProviderList.tsx (larger sidebar variants). Two copies of the same paths means any brand-asset update has to be made in two places — classic drift bait. This was deliberately deferred out of plans/06052026-mission-control-integration-icons to avoid mixing a refactor into a feature; this plan is that refactor.

What

Define each logo's SVG exactly once in a new neutral asset module integrationLogos.tsx, where every logo accepts a className so the caller owns sizing. Both consumers import from it. Done when the SVG geometry lives in one place, both call sites pass their original size classes, and the rendered output is pixel-identical in the compact badge row and the provider sidebar.

Context

Relevant Files

  • src/components/Admin/integrations/integrationLogos.tsx - NEW. Single source of truth for the four raw brand SVGs.
  • src/components/Admin/integrations/ActiveIntegrationLogos.tsx - compact badge row (Websites tab + Mission Control card). Drops its inline SVGs, imports shared ones at h-3.5/h-4.
  • src/components/Admin/integrations/IntegrationProviderList.tsx - provider sidebar (IntegrationsTab). Drops its inline SVGs, imports shared ones at w-4.5/w-5/w-4.

Patterns To Follow

  • Named exports for non-page components; arrow-function components matching existing logo style.
  • Automatic JSX runtime (no import React); only import type when a React type is referenced — same as the existing files.
  • Caller-owns-sizing via a className prop, mirroring how icon primitives are sized across the app.

Reference file: ActiveIntegrationLogos.tsx - closest analog; the extracted components are lifted verbatim from its SVG definitions.

Constraints

Must

  • Preserve every existing export of ActiveIntegrationLogos.tsx (ActiveIntegrationLogos, IntegrationPlatform, ActiveIntegration).
  • Preserve the sidebar's larger per-logo sizes exactly (HubSpot w-4.5, Rybbit/Clarity w-5, Google w-4).
  • Preserve the badge row's compact sizes exactly (HubSpot/Google h-3.5, Rybbit/Clarity h-4).

Must Not

  • No new dependencies.
  • Don't shrink or restyle the sidebar logos.
  • Don't touch the duplicate IntegrationPlatform in api/integrations.ts (separate concern).
  • No behavior change — pure extraction.

Out Of Scope

  • De-duplicating the IntegrationPlatform union that also lives in api/integrations.ts.
  • Any visual restyle of either consumer.

Risk

Level: 1 (Suggestion / low risk)

Risks Identified

  • Accidentally changing a size class and altering rendered output. Mitigation: copy the exact class strings from each original call site; verify in the browser.
  • Dropping xmlns from the sidebar SVGs. Mitigation: none needed — xmlns is a no-op for inline JSX SVG; React renders in the SVG namespace by tag.

Blast radius: ActiveIntegrationLogos consumed by pages/admin/WebsitesList.tsx, components/Admin/mission-control/OrganizationMissionCard.tsx, and api/admin-mission-control.ts (type only). IntegrationProviderList consumed by components/Admin/IntegrationsTab.tsx. None reference the raw logo components directly, so extraction is internal.

Pushback

  • None material. The only adjacent cleanup (the duplicated IntegrationPlatform union) is intentionally left out to keep this a single-purpose refactor.
