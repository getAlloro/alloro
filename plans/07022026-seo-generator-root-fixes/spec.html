Why

Two real generator bugs were discovered during plans/07022026-seo-metatag-fixes and plans/07022026-seo-full-coverage but only worked around at the call sites of two one-off scripts — every other caller (the admin UI's per-page/per-post "Generate" and "Generate All" buttons, and the bulk-generate worker) still has them. Until they're fixed at the root, any admin click can re-introduce the exact data corruption those two plans cleaned up across ~380 posts: fabricated canonical URLs, and a silent body-content rewrite whose recovery snapshot has never once succeeded (0 non-null previous_content across all 553 posts platform-wide — the snapshot write crashes on a column-type mismatch, which also makes the whole generation call fail for any post where the GEO section produces a recommendation).

What

Three root fixes in the shared generation pipeline, protecting every current and future call path:

  • Canonical is never LLM-authored again. The prompt stops asking for it; a deterministic derivation (post: /{post_type.slug}/{post.slug}; page: its real path) is applied in one shared seam that every caller flows through.
  • GEO auto-apply: keep + fix + explicit opt-out (owner decision 2026-07-02, option A): the approved 0.0.143 behavior stays default-on for the admin UI and worker, the broken snapshot gets fixed so recovery genuinely works, the side effect gets explicit naming/logging, and callers get an applyGeoContent flag to skip content mutation (formalizing what the session scripts did by bypassing the wrapper).
  • previous_content column type corrected from jsonb to text via migration — the column stores raw HTML, every reader already treats it as a string, and it is provably all-NULL in prod, so the change is zero-data-risk.

Done when: a "Generate All" run through the real controller path produces a correct deterministic canonical and a working post-content snapshot, the metadata-only flag verifiably skips content mutation, and the one-generation-engine claim is confirmed and documented.

Constraints

Must

  • Canonical derivation applies on every path that writes seo_data: UI single-section, UI generate-all, bulk worker, both scripts.
  • applyGeoContent defaults to true (behavior-preserving for UI/worker); scripts pass false.
  • Snapshot fix proven with a real-HTML post end to end before declaring done.
  • Migration is reversible (text → jsonb down is safe precisely because the column is all-NULL; assert that in the migration).

Must Not

  • Do not change what the GEO recommendation writes (prepend-as-paragraph stays as approved in 0.0.143).
  • Do not regenerate or touch any existing prod seo_data — data is already correct; this plan is code-only plus one column-type migration.
  • Do not remove canonical_url from stored seo_data shape (renderer reads it verbatim) — override its value, don't drop the key.

Out Of Scope

  • Renderer-side canonical derivation (separate plan: 07022026-renderer-seo-hardening).
  • A preview/confirmation UI for auto-applied content — the opt-out flag is the control this round; a review UI is a future product decision.

Risk

Level: 3 — shared pipeline used by every org's admin SEO tooling, plus a schema migration.

Risks Identified

  • Level 2 The canonical override needs the entity's slug/type — the worker path merges sections without entity context today. Mitigation: derive once (helper takes entityId + entityType), call it at each save seam; T1's call-path map is the checklist that none is missed.
  • Level 2 Fixing the snapshot un-breaks GEO auto-apply for posts — meaning post-content rewrites that have been silently failing will START succeeding for UI/worker callers. That is the owner-chosen outcome (option A), but it is a behavior activation, not just a bug fix. Mitigation: explicit log line on every auto-apply, verified restore path (previous_content populated and readable in the admin editor), called out in the changelog.
  • Level 1 Migration on a live table. Mitigation: all-NULL column (verified 0/553 in prod), ALTER ... TYPE text USING previous_content #>> '{}' is instant; dev first per deployment path.

Blast radius: four SeoController endpoints (every org's admin SEO tools), seoBulkGenerate worker, both session scripts, the posts editor's restore surface, service.seo-generation.test.ts.

Pushback

  • None on direction — but note the alternative to the prompt edit (keep asking the LLM and only override in code) was rejected: leaving the instruction in the prompt invites drift and wastes tokens on a field we discard. Belt (prompt stops asking) and suspenders (code overrides regardless) together.
