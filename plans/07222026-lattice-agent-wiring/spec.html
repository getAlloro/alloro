Why

Alloro's product agents each read a single instruction file (src/agents/{agent}.md) through loadPrompt(). That is the only channel by which knowledge reaches a running agent. Everything outside that channel — the knowledge lattice, the sentiment lattice, the compiled master rubrics — is read by no code (git grep lattice in src/ returns zero). A 845-line knowledge lattice that nothing loads changes no agent's behavior. It is a well-bound dusty book.

This is also the fix for a second, larger problem: decisions that live as prose get re-litigated every session. The durable fix for both is one move — encode a principle where it is enforced (a loader the agent can't skip), not where it is re-read (a doc). This spec builds the first, smallest instance of that: a validated lattice fragment, wired into one agent's prompt, proven by a test that fails if the wiring is removed. It is the reference implementation of the repeatable path {new knowledge} → encoded → enforced → live.

Done when: one pilot agent's assembled system prompt provably contains a validated master's rubric, a test mutation-proves the wiring (delete the mapping → suite fails), and adding the next master is a three-line change (fragment + mapping row + test).

What

A small, backward-compatible composition layer over the existing prompt loader, plus the first curated fragment and the registry that maps agents to fragments.

library (held outside this repo) ── validated + distilled ──▶ src/agents/lattice/*.md (the shipped loadout: canon-conformant fragments) │ AGENT_LATTICE_LOADOUT { "auditAgents/gbp/TrustEngagement": ["cialdini-honest-influence"] } │ loadAgentPrompt(agentPath) = loadPrompt(agentPath) + mapped fragments ──▶ systemPrompt ──▶ live agent │ test: assembled prompt CONTAINS the fragment (remove mapping → FAIL)

Pilot pairing: Cialdini (honest influence — surface real proof, strip manufactured signals) → auditAgents/gbp/TrustEngagement (the GBP trust/engagement audit). Chosen because the source row already exists in the lattice (least authoring risk), the fit is exact, and it does not collide with the active CtrHypothesis Brick work.

  • src/controllers/admin-websites/feature-utils/util.seo-section-runner.ts:105-106 — the reference analog for composition. Already loads a base prompt plus a mapped section fragment and combines them. The wiring mirrors this exactly; it invents no new pattern.
  • src/agents/auditAgents/gbp/TrustEngagement.md — the pilot agent instruction file (read before touching its call site).
  • The GBP audit caller that runs TrustEngagement — the single call site to switch from loadPrompt(...) to loadAgentPrompt(...) (located during execution via git grep "auditAgents/gbp/TrustEngagement").
  • The external lattice library (Cialdini row) — the library source, held outside this repository. Not read at runtime; distilled by hand into the shipped fragment. Status provisional / drift-flagged — only validated, canon-conformant content crosses into the product.

Patterns To Follow

  • Prompt composition = base + fragments, mirroring util.seo-section-runner.ts. Fragments are loaded through the existing loadPrompt("lattice/{key}") — src/agents/lattice/ sits under the same AGENTS_DIR, so no new file-reading code.
  • The agent→fragment mapping is a single named const registry (§4.2), the one place that declares which principle wires into which agent.
  • Backward-compatible: an agent with no registry entry gets output byte-identical to loadPrompt(). Blast radius is exactly the mapped agent.

Reference analog: §6.1 src/controllers/gbp-automation/ for structure; util.seo-section-runner.ts:105-106 for the composition shape.

Code Constitution Articles this work touches

  • §6.2 Every file has a home — new fragment dir src/agents/lattice/, registry beside the loader.
  • §2.2 Functions under ~50 lines — loadAgentPrompt() stays small.
  • §4.2 No magic strings — the loadout mapping is a named registry const.
  • §3.1 / §3.2 Error handling — reuse the loader's existing throw-on-missing-file; a mapped fragment that doesn't exist must fail loud, never silently drop.
  • §20 Tests — the mutation-proven composition test is the enforcement artifact, not an afterthought.

Constraints

Must

  • Keep loadPrompt() unchanged and backward-compatible; add composition alongside it.
  • Ship the fragment inside the alloro repo (src/agents/lattice/). Deploy has no access to the external library — a runtime read of it would break in prod.
  • Author the fragment fresh and canon-conformant: no retired vocabulary, no ad/campaign/impressions-as-vanity language, no guarantees (Value #6). Validate against START HERE canon before it ships.
  • Fail loud if a mapped fragment key is missing (reuse the loader's existing throw).
  • Wire exactly ONE pilot agent. Prove it. Generalize later.

Must Not

  • Copy the provisional lattice wholesale into any agent prompt (drift → live agents).
  • Read any external library or artifact store at runtime.
  • Modify more than the one pilot call site.
  • Add a dependency, or hand-fold the same principle into multiple agent files (that recreates the duplication this replaces).
  • Touch CtrHypothesis (active Brick surface).

Out Of Scope

  • Wiring the other agents (Schwartz→CRO/SeoGeneration, Sheridan→WebsiteAnalysis) — trivial follow-ups once the mechanism is proven, each a fragment + mapping row + test.
  • Landing Sheridan/Schwartz/Cialdini rows into the git lattice library (human-staked, separate; Notion is being retired).
  • Any runtime measurement of whether the wired principle improves agent output quality (that needs the measurement build; this spec only proves the principle reaches the agent).
  • Housekeeping on the external master-rubrics doc (done once its net-new rows land in the library).

Risk

Level: 2 — Concern. Scoped, backward-compatible, one call site.

Risks Identified

  • Drift injection — the lattice is drift-flagged; a stale fragment would push retired positioning into a live agent. Mitigation: fragments are authored fresh and canon-checked, never copied from the provisional mirror; the fragment is the rubric (heuristic + falsifier), not the essay.
  • Token cost — appending a fragment grows every call's system prompt. Mitigation: fragments are tight; measure the token delta on the pilot and record it before generalizing.
  • Silent no-op — a mapping that points at a missing/misnamed fragment could degrade to the base prompt unnoticed. Mitigation: reuse the loader's throw-on-missing; the test asserts the fragment's signature line is present in the assembled prompt.
  • Prod cache — loadPrompt caches per-key in prod. Composition must cache correctly or read fresh; confirm no stale-fragment bug across the base+fragment keys.

Blast radius: loadPrompt() has ~15 callers; the composition helper is additive and only the single pilot call site changes behavior. All other callers keep calling loadPrompt() and are byte-for-byte unaffected.

Pushback — the design decision, and why this path

  • Option A (rejected): loadPrompt appends lattice rows to all agents. Over-broad, injects drift everywhere, uncontrolled token cost.
  • Option B (rejected): hand-fold heuristics into each agent .md. No single source; the same principle duplicated across N files — the exact duplication this is meant to end.
  • Option C (chosen): curated fragments + a mapping registry + a composition helper. One source per principle, composed at load, canon-gated, test-enforced, generalizes one mapping-row at a time. This is what Musk (simplest that works), sequential-build (pilot one, prove, then scale), and the drift caveat (curated, not raw) jointly point to.
