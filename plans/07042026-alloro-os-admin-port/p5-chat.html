Why

Chat is the second pillar the user named ("chat as another tab within that tab"): ask a question, get an answer grounded in the team's own documents, with citations you can click — or an honest refusal when the KB has nothing.

What

Backend: conversation CRUD (list with last-activity preview, delete cascades), context-document attach/detach (manual @-attach + AI-retrieved), and the streaming message endpoint — persist user message → async auto-title on first message → build history → embed query + retrieve (P4 service, respecting attached context) → SSE progress events (Searching the knowledge base… / Composing a grounded answer…) → Gemini token stream → done event with message id + citations → persist assistant message with citations jsonb. Grounded-refusal system prompt: answer only from retrieved context; cite only chunks actually used; say so plainly when nothing relevant exists. Frontend: the Chat sub-tab — conversation sidebar, message thread (user right-aligned Jakarta, assistant Spectral), thinking indicator until first token, stream-following scroll with jump-to-latest pill, citation chips (bordered pills: doc title · heading) that navigate to /admin/os/doc/:id, input with @-mention document picker and context chips.

Risk (phase-local)

  • R1 (master): installed @google/genai 1.43 vs OS code on 2.x — verify streaming API shape first; if 1.43 lacks parity, bump the package (Rev entry) or fall back to the Anthropic seam (master Pushback). Gate includes a live one-shot smoke.
  • Half-written assistant messages on disconnect. Mitigation (ported): persist assistant message only on completed stream; client AbortSignal on unmount; user message already persisted so retry is natural.
  • Citation hallucination. Mitigation (ported): citations built server-side from the retrieval result set only — never parsed from model output.
  • (Rev 1) Apache buffering kills streaming: the catch-all ProxyPass / on dev/prod does not flush — minds chat needed a dedicated flushpackets=on Location block to stream. Mitigation: T6 mirrors that block for /api/admin/os/chat/ on both vhosts before UI sign-off; localhost testing bypasses Apache, so this only shows up through the public URL — test through dev.getalloro.com.
