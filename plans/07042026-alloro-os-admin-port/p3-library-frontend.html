Why

This phase makes the OS tab genuinely usable: browse, read, write, version. It is also where the "match the Alloro app light mode" requirement is actually delivered — every component is recreated on Alloro tokens, not copied OS markup.

What

Library page (list / grouped-by-category / folder-tree views with @dnd-kit drag-move, filters, sort), DocumentRead 3-pane (doc rail · 70ch Spectral reading column with TOC · right rail with History and placeholders for Related/Comments), DocumentEdit (TipTap + markdown storage, debounced draft autosave, lock banner + heartbeat, publish modal with note), version history with diff + restore, and Trash (restore / permanent-delete with confirm). Empty/loading/error states designed for every surface (skeletons matching row shapes — never a centered spinner).

Context

Design contract (D13 + OS DESIGN.md, re-expressed in Alloro tokens)

  • Surface: bg-alloro-bg (#FAF8F3) page, bg-alloro-surface white only for genuinely bounded objects (modals, editor pane). Lists are hairline-divided rows (--color-line-soft), NOT card grids.
  • Type: Spectral (font-display) for document titles + reading body only; Plus Jakarta (font-sans/font-body) for all chrome; JetBrains Mono (font-mono) for timestamps/status/counts with tabular-nums.
  • Accent #D66853 (alloro-orange) for primary actions, active states, selection wash (--color-accent-soft) — never decoration. Status dots: indexed alloro-success · processing amber pulse (--color-amber) · failed alloro-danger · archived muted.
  • Radius: controls 9px, inputs 8px, surfaces 12px. Motion: 150–250ms ease-out, state-conveying only; honor prefers-reduced-motion.
  • Icons: lucide-react only.
