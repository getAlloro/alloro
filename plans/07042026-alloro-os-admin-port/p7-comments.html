Why

Completes the "entire features minus tasks" scope: lightweight discussion on documents. The OS unified comment/task table is deliberately split here — comments port, task conversion does not (pmtool owns tasks; master D-scope).

What

Threaded comments per document (one nesting level of replies like the OS UI), markdown body, version_tag recording the live version at comment time, edit + tombstone delete ("Comment deleted" placeholder keeps thread shape), rendered in the DocumentRead right rail with composer, relative mono timestamps, and author identity from admin users. Explicitly absent: is_task/assignee/due-date/resolve controls, mentions-notifications (no notification infra by master scope).

Context

Port sources

  • alloro-os/backend/src/services/comments/CommentService.ts + models/CommentModel.ts (strip task logic), frontend/src/components/comments/{CommentThread,CommentForm}.tsx (strip TaskItem/TaskControls).

Files to create

  • src/models/OsCommentModel.ts, feature-services/OsCommentService.ts, AdminOsCommentsController.ts, route additions (GET/POST /documents/:id/comments, PATCH/DELETE /comments/:id — author-only edit/delete enforced server-side §5.4).
  • frontend/src/components/Admin/os/comments/{OsCommentThread,OsCommentItem,OsCommentComposer}.tsx; hook useAdminOsComments.ts; replace the P3 placeholder section in the read right rail.
  • Tests: src/__tests__/os-comments.test.ts (threading, tombstone, author-only edit, version_tag capture).
