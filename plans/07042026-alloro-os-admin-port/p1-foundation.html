Why

Every later phase needs the same substrate: the dedicated os Postgres schema, config, the gated /api/admin/os domain, the BullMQ queue seam, and a visible OS tab shell. Landing it once keeps P2–P7 purely additive.

What

After P1: the migration is applied (schema os with 16 tables on dev, Rev 2), /api/admin/os/ping and /api/admin/os/users respond only to super-admins, the worker registers OS queues (stub processors + live lock-reaper), and the admin shows an OS tab rendering a warm-paper shell with Library/Chat sub-tabs and honest empty states. Master spec decisions D1–D14 all apply; this file only adds phase detail.
