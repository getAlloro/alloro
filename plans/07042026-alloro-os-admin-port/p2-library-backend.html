Why

The Library is the product core. A complete, tested HTTP API lets P3 build UI against real behavior and lets P4 hook indexing into an already-correct publish lifecycle.

What

Full document lifecycle over /api/admin/os: create (status processing + ingest enqueue), draft autosave, publish (draft → version N+1 → current_version_id → re-ingest enqueue), rename, metadata edit (category/tags/owner with meta_locked), folders (tree + move), categories registry, archive → trash → restore → purge (queued), edit-locks (acquire/heartbeat/release + conflict semantics), lexical search (weighted tsvector + ts_headline snippets), version list/diff/restore, and activity logging on every state change. All list endpoints paginate per §11.6.

Key behaviors (port faithfully)

  • Publish is transactional (§10.5): insert version, bump current_version_id, clear draft, enqueue re-ingest — all-or-nothing.
  • meta_locked: once category/tags are human-edited, re-ingest may update summary only.
  • Locks: acquire fails with OS_LOCK_HELD (409-mapped) if another live lock exists; heartbeat extends expires_at by OS_LOCK_TTL_SECONDS; owner release is idempotent.
  • Purge removes chunks/versions/drafts via cascade + S3 asset keys via queue job (job body carries keys; §21.1 idempotent).
  • tsv weights: title A · content B · summary C · tags C (rebuild helper shared with P4 ingest).
  • Slug: unique, regenerated on rename with collision suffix.

Constitution

§7.1–§7.4, §8.1–§8.4, §9.1–§9.3, §10.1–§10.5, §11.2, §11.6, §2.4 (split controllers), §20.1–§20.2. Analog: gbp-automation service/controller split.

Risk (phase-local)

  • Raw SQL for tsvector/ts_headline — keep parameterized inside models only (§10.2); reuse the vetted-literal lesson from the telemetry GROUP-BY incident: build shared SQL fragments as exported constants, never interpolate user input.
  • Publish/lock race: guard publish with an active-lock ownership check; covered by tests.
