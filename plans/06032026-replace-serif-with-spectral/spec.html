Replace Serif With Spectral

Status
Completed
Plan Folder
plans/06032026-replace-serif-with-spectral
Size
Medium
Risk
Level 2

Why

The active frontend app still mixes Fraunces and Literata for serif display surfaces. The request is to standardize every live frontend serif stack on the Google Font Spectral.

What

Load Spectral from Google Fonts, replace the app display serif token with Spectral, and update hardcoded frontend serif constants/usages that currently point to Fraunces or Literata.

Context

Relevant Files

  • frontend/index.html - Google Font loader for the SPA.
  • frontend/src/index.css - global Alloro typography tokens and display utility.
  • frontend/src/components/dashboard/focus/*.tsx - hardcoded Focus dashboard serif number stacks.
  • frontend/src/components/Admin/minds/*.tsx - hardcoded chat serif reading stacks.
  • frontend/src/components/dashboard/RankingsDashboard.tsx - direct SVG font family assignment.

Patterns To Follow

  • Keep the existing font loading pattern in frontend/index.html.
  • Keep direct style constants where the surrounding file already uses local constants instead of introducing a new abstraction.

Reference file: frontend/src/index.css - existing display typography token.

Constraints

Must

  • Use Spectral as the active frontend serif/display font.
  • Preserve Plus Jakarta Sans, Inter, and JetBrains Mono where they are already used.

Must Not

  • Do not touch unrelated dirty worktree changes.
  • Do not rewrite historical plan artifacts or generated snapshots.
  • Do not introduce a new dependency.

Out Of Scope

  • Email serif stacks and historical specs/snapshots are out of scope for this frontend font swap.

Risk

Level: 2 - Concern

Risks Identified

  • Hardcoded font stacks can drift from the global display token. Mitigation: replace every live frontend Fraunces/Literata serif declaration found by search.
  • Touching backend email or historical generated artifacts would create broad churn with weak user value. Mitigation: scope to active frontend runtime files.

Blast radius: active SPA typography, Focus dashboard metric cards, Rankings gauge text, Minds chat surfaces, and overview/section title comments.

Pushback

  • This should eventually become one token, not repeated constants. Future-us will hate another hardcoded font sweep if these components keep local font stacks.
