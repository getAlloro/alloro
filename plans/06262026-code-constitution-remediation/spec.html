Why

The current convention check is blocked by hard backend violations and a finite set of frontend and backend advisory violations. We need the codebase to move closer to the numbered constitution instead of carrying avoidable drift.

What

Remediate the convention-runner findings that are local and mechanical: backend file size, backend raw logging strings, query placement, frontend API boundaries, unsafe HTML handling, token access, and import-boundary findings. Keep behavior stable.

Context

Relevant Files

  • src/controllers/pms/PmsController.ts - currently exceeds the backend file-size ceiling.
  • src/controllers/googleauth/utils/templates/successTemplate.ts and src/lib/logger.ts - contain backend console. strings caught by the checker.
  • src/app.ts, src/routes/billing.ts, and src/validation/billing.schemas.ts - are reported by raw-query checks.
  • frontend/src/api/pm.ts, frontend/src/utils/publishLint.ts, and frontend/src/utils/templateRenderer.ts - are reported by frontend raw HTTP checks.
  • frontend/src/components/Admin/leadgen/ and frontend/src/components/pm/RichTextEditor.tsx - are reported by HTML injection checks.

Constitution Articles

  • §2.4 file-size ceiling for living source files.
  • §7.4, §9.1, and §10.2 backend layering, logging, and query-placement rules.
  • §12.1, §12.4, §14.2, §17.4, and §17.5 frontend API, import, HTML, and token-boundary rules.

Constraints

Must

  • Preserve existing behavior and public API shapes.
  • Use established controller, route, model, API helper, and React patterns.
  • Keep fixes narrow to convention findings.

Must Not

  • Do not revert unrelated patient-journey or market-intelligence work in the dirty tree.
  • Do not add new dependencies.
  • Do not make product behavior changes just to satisfy a checker.

Risk

Level: 3

Risks Identified

  • Some convention findings are heuristic and may be false positives. Mitigation: inspect each reported file before changing it and avoid behavior changes where a local naming or helper fix is enough.
  • The working tree is dirty with unrelated work. Mitigation: touch only convention-remediation files plus this plan folder.

Blast radius: PMS routes, Google OAuth success screen, logger documentation, billing routes/validation, frontend publish utilities, admin auth shell, auth pages, PM editor, and leadgen submission detail rendering.
