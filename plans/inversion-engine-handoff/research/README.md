# Alloro Research Library

Deep research, stored ONCE so it is never re-derived. **Before commissioning research on a company
or topic, CHECK HERE FIRST.** If it exists, amend/update it; do not re-run it from scratch, that is
the waste this library exists to end. Every file carries its sources (URLs) and grades (PROVEN vs
INFERRED) so it can be trusted and updated.

## The rule this library enforces
- **Research-store-first.** Before a research battery on a domain, check this library (and canon).
  Re-deriving stored research is the same waste as re-deriving locked canon.
- **Store on completion.** When research finishes, its synthesized, sourced findings land here as a
  file, not in a chat that evaporates.
- **Amend, don't re-run.** Update the file as the world changes; keep the source + date so staleness
  is visible.

## The skill that runs this
**`alloro-research`** (in the alloro-skills repo) owns this discipline: *check-first* before a research
battery, *store-on-completion* after. Its tool is **`scripts/store-research.py`** — it sweeps every
session's subagent transcripts into `raw/` and rebuilds the INDEX (idempotent, re-run anytime). So:
grep `raw/INDEX.md` before you commission research; run `python3 scripts/store-research.py` after.
