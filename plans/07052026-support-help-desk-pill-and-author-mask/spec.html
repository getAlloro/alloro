Support Help Desk — Pill Placement & Author Mask

Status
Completed
Plan Folder

Size
Small
Risk
Level 1

Why

Two client-facing polish issues on the Help desk ticket detail. (1) When a ticket title is long, the status pill ("Waiting on you") is squeezed into a narrow right-hand column and wraps mid-phrase across two lines. (2) The message thread shows the real name of the Alloro team member who replied; the client should just see "Alloro" so individual staff identities are not exposed to clients.

What

On the client Help desk ticket detail: the status pill moves above the ticket ID/title (stacked, left-aligned) and never breaks mid-phrase. In the client message thread, non-client (staff) authors render as "Alloro" instead of the person's name. The admin support panel is unchanged — admins still see the real replier name.

Context

Relevant Files

  • frontend/src/components/support/SupportTicketDetail.tsx - client ticket detail; owns the header (pill + title) and renders the thread.
  • frontend/src/components/support/SupportMessageThread.tsx - shared thread; renders each message's author name. Used by BOTH client detail and admin panel.
  • frontend/src/components/support/SupportStatusBadge.tsx - the status pill.
  • frontend/src/components/Admin/support/AdminSupportTicketPanel.tsx - admin panel; also renders SupportMessageThread. Must keep real names — left unchanged (no new prop passed).

Patterns To Follow

  • Constitution §1.3 / §13.3 — boolean prop reads as a question/state: maskStaffName.
  • §13.2 lean presentational components; §17.1 no console.*; §17.2 no any; §12.3 files already in the components/support/ feature folder.

Reference file: frontend/src/components/support/SupportMessageThread.tsx - existing optional-prop and Tailwind conventions matched in place.

Constraints

Must

  • Mask staff name on the client thread only, via an opt-in prop.
  • Keep the admin panel showing real names.
  • Pill above the title; pill text never wraps.

Must Not

  • No hard-coded global mask in the shared component.
  • No new dependencies; no refactor of unrelated support code.
  • No backend/API change — display-only.

Out Of Scope

  • Ticket list card layout, composer, attachments, status logic.
  • Any backend author-name storage change (names still stored; only client display is masked).

Risk

Level: 1 (Suggestion / low risk — display-only, no data or auth change).

Risks Identified

  • SupportMessageThread is shared by client and admin. Mitigation: mask is an opt-in prop (maskStaffName, default false); only the client detail passes it, so the admin panel is untouched.

Blast radius: two consumers of SupportMessageThread — SupportTicketDetail.tsx (client, opts in) and AdminSupportTicketPanel.tsx (admin, unchanged). SupportStatusBadge consumers: same two surfaces; adding whitespace-nowrap is safe everywhere.

Pushback

  • Masking is display-only — the real author name is still in the API payload/DB. If true confidentiality from the client were required, the backend would have to stop sending it. For this ask (UI presentation), display masking is the correct, minimal change.
